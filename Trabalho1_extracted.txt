Enunciado 
Introdução 
 O objetivo deste trabalho é desenvolver uma loja virtual em Java, com a qual 
o utilizador interage através da consola. O sistema tem 2 tipos de utilizador: 
Cliente e Dono da Loja. 
• O Cliente pode consultar, comprar e devolver produtos da loja. 
• O Dono da Loja consegue mudar o nome da loja e gerir os produtos. 
O programa deve ter apenas 1 loja e pode ter vários clientes. A loja apenas tem 1 
dono e pode ter vários produtos. 
O aluno é livre de escolher o tema da loja e os respetivos produtos, desde que a 
implementação cumpra todos os requisitos definidos no presente enunciado. 
 
Loja 
 A loja, representada pela classe Store, tem um nome, exatamente 1 dono e 
um inventário, que representa o stock da loja. 
A loja deve ter 2 vistas: vista do cliente e vista do dono. Ambas as vistas devem 
ter o nome da loja, o utilizador atual (com nome, se é cliente ou dono, e saldo), a 
lista de produtos e a lista de ações possíveis para o utilizador atual. 
A lista de produtos deve conter um número (índice), começando em 1. Deve 
também conter o stock atual e o preço por unidade do produto. 
A lista de ações deve conter para cada ação uma descrição do que ela faz. Para 
executar uma ação o programa deve ler do utilizador o nome da ação e em seguida, 
se assim se aplicar, pedir os dados necessários. Certifique-se que são apresentados 
os devidos erros caso o utilizador forneça dados inválidos, por exemplo índice de 
produto inválido. Em caso de dados inválidos, a ação deve ser cancelada e a devida 
mensagem de erro deve ser apresentada. Caso a ação termine com sucesso, deve ser 
apresentada a devida mensagem de sucesso. 
 
 
 
 
 
As ações que um cliente pode executar são: 
• store 
• buy 
• inv 
• return 
• client 
• owner 
• register 
As ações que o dono da loja pode executar são: 
• store 
• add 
• remove 
• edit 
• stock 
• rename 
• client 
• register 
O programa deve impedir o utilizador de executar uma ação que o tipo de utilizador 
não possa executar.

Inventários 
Um inventário, representado pela classe Inventory, permite armazenar e gerir 
produtos. Um inventário deve ser capaz de armazenar até um máximo de 100 
produtos. 
O inventário deve conter métodos para adicionar, remover e obter produtos. 
Utilize esta classe para armazenar os produtos da loja e os produtos que cada 
cliente comprou. 
 
Interface da Loja 
Crie uma classe StoreInterface que pede e lê dados do utilizador, usando uma 
instância da loja para executar as ações pedidas pelo utilizador. 
 
Requisitos 
• O trabalho deve ser implementado em Java, com interação do utilizador 
através da consola. 
• Deve utilizar apenas os conhecimentos e conteúdos lecionados até ao 
momento nas aulas. 
• O código deve seguir boas práticas de programação orientada a objetos, 
utilizando conceitos como classes, herança, encapsulamento e polimorfismo, 
tal como dado nas aulas. 
• A implementação deve ser modular, organizada e de fácil manutenção, 
permitindo expansão futura do sistema. 
• A única classe que deve ler input é a classe StoreInterface. 
• As únicas classes que devem imprimir na consola são as classes Store e 
StoreInterface. 
• Quando o programa é executado, deve criar uma loja e um dono para a 
mesma. Deve também criar alguns clientes e adicionar alguns produtos 
iniciais. O programa deve começar com o primeiro cliente selecionado. 
• A classe Main deve conter apenas a inicialização da loja, dono, clientes, 
produtos e interface da loja. 
 
Notas 
• Adicione todas as classes, atributos e métodos que achar necessário. 
• Adicione comentários onde achar necessário, para ajuda própria e para 
quem for ler o código.
